# Web Viewer Prototype

This directory contains a polished static prototype for the first web-facing
slice of Tarmac Trace Utilities.

## What works today

- Load a call tree JSON export generated by `tarmac-calltree-json`
- Explore nested calls in a browser
- Search functions by name or address
- Inspect entry, exit, and duration metadata
- Use the bundled quicksort sample for a fast demo

## Run the prototype

From the repo root:

```sh
python3 -m http.server 8123
```

Then open:

```text
http://localhost:8123/web/calltree-viewer.html
```

Use one of these flows:

- Click **Load bundled sample**
- Drag in `tests/calltree-quicksort-symbols.json.ref`
- Generate a fresh export with `tarmac-calltree-json`

## Fresh export command

With a normal CMake build:

```sh
./build/tarmac-calltree-json \
  --index build/quicksort.tarmac.index \
  --image tests/quicksort.elf \
  tests/quicksort.tarmac \
  -o quicksort.calltree.json
```

## How this maps to the long-term architecture

Today the viewer reads a JSON file directly. The future thin API can keep the
same data contract and replace file loading with HTTP requests.

## Validation notes

Current automated coverage in the repo includes:

- symbol-aware JSON export regression on the sample quicksort trace
- no-symbol JSON export regression on the same trace
- Python JSON parsing validation for the exported schema

Manual smoke test for the prototype UI:

1. Generate `quicksort.calltree.json`
2. Serve the repo with `python3 -m http.server 8123`
3. Open `web/calltree-viewer.html`
4. Load the JSON file
5. Search for `quicksort`
6. Select a deep nested node and verify the detail pane updates
